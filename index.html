New Custom Object Checklist

Setup

Object name is a singular noun

Object description contains details on business use for the object as well as any major assumptions the data model makes

Reports are enabled

Sharing is enabled

Search is allowed if the object contains any external IDs

Auto-number only active for junction objects or objects where there is no semantic name

If auto-number is used, the format PREFIX-{0000} is used, starting with number 1. PREFIX should be no longer than four letters, and pronounceable.

Naming strategy (user input or autonaming) determined and documented

Fields

Field label is expressive. It may be appropriate to use an objects name in the field label (e.g. Enrollment Start Date) which can be omitted in the field name (e.g. Start_Date__c)

All fields have description.

Field help text is used appropriately but not excessively (help text such as "The date a child's enrollment started" is unhelpful)

Page Layouts

Mini page layout is configured

Compact Page Layout is configured and assigned

Fields in the compact page layout do not appear in the first two fields on the page layout, if it can be helped (avoid Cory Davis, Cory Davis, Cory Davis Household, Cory Davis pattern).

All 2-column page layout sections use top-bottom tab order.

Chatter Feed Tracking is enabled for object (with no fields tracked preferred) in order to display the publisher.

Field tracking in the feed is not used for field history/audit tracking purposes (Field History is used instead)
